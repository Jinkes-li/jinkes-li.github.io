  • Mysql 多表联查两种方式
-- 方式一

-- inner join 内连接
/*
使用内连接即使将两张表的结果值拼起来，如果条件对应不上则不显示该条记录
*/

select * from emp inner join dep on emp.id = dep.id

-- left join 左连接
/*
在使用时和内连接一样，不同的是以左表为基准，如果右表的条件没有匹配则显示NULL
*/

select * from emp left join dep on emp.id = dep.id


-- rigth join 右连接
/*
以右表为基准，如果左表的条件位匹配则使用NULL填充
*/
select * from emp rigth join dep on emp.id =dep.id

-- union 全链接
/*
全连接就是使用左链接和右链接查询之后的数据合并到一起(查询的数据未匹配则使用NULL填充)
*/
select * from mep left join dep on mep.id = dep.id
union
select * from emp rigth join dep on mep.id = dep.id

-- 多表联查的第二种方式子查询
/*
子查询 指的就是将上一个SQL查询的语句作为条件 在本条SQL中使用
*/
-- 下列语句()中就是查询的用户id 作为条件在本条SQL中使用
select name from dep where id=(select dep_id from emp where name='jason')

  • MySQL 事务

mysql事务主要用于复杂性操作，为了保证数据的安全性，引发出的mysql事务，事务具有ACID四个特性

  1. 原子性 原子性是保证了数据的不可分隔性，在一个事务中，要么全部成功，要么全部失败，如果在执行过程中发生错误，事务会被回滚到事务开始之前的样子

  2. 一致性 事务在开始和事务结束之后，数据的完整性没有被破坏，这表示所有的资料完全符合预期

  3. 隔离性 数据库允许并发事务同时对数据继续兑取和修改的功能，隔离可以防止多个事务的交叉执行导致的数据不一致问题，隔离性中有4个隔离级别

    1. 读未提交，即事务修改单没有提交，可能引发脏读问题
    2. 读提交，也可以叫不可重复读，在一个事务继续修改时，对其他事务时不可见的
    3. 可重复读 可重复读是mysql中使用的默认隔离级别，可以解决掉数据的脏读问题，但是又带来了幻读，幻读指的是，当某个事务在去某个范围内的记录时，另一个事务在在这个范围内插入了新的记录，当前事务再次取值就会产生幻行，这个问题InnDB和XtraDB通过多版本并发控制(MVCC)及间隙策略解决该问题
    4. 串行化 强制事务串行

  4. 持久性 事务处理结束后，对数据的更改是永久的 即存储在硬盘

  1. 聚集索引(primary key)
  2. 辅助索引(unique, index)
  3. 覆盖索引(需要查询的字段值与查找字段属于同一位置)
  4. 非覆盖索引(需要查询的数据值与查找字段不在一个位置)

  1. <!DOCTYPE html> 声明为HTML5文档

  5. <body></body> 之间的文本是网页可见的主体内容

    • <meta>元素可提供有关页面的元信息（mata-information）,针对搜索引擎和更新频度的描述和关键词。
    • <meta>标签位于文档的头部，不包含任何内容。
    • <meta>提供的信息是用户不可见的。

meta标签的组成：meta标签共有两个属性，它们分别是http-equiv属性和name 属性，不同的属性又有不同的参数值，这些不同的参数值就实现了不同的网页功能。

  1. http-equiv属性：相当于http的文件头作用，它可以向浏览器传回一些有用的信息，以帮助正确地显示网页内容，与之对应的属性值为content，content中的内容其实就是各个参数的变量值。
<!--指定文档的编码类型-->
<meta http-equiv="content-Type" charset="utf8">
<!--等待两秒跳转对应的地址-->
<meta http-equiv="refresh" content="2;URL=http://www.baidu.com">
<!--告诉IE以最高级模式渲染文档-->
<meta http-equiv="x-ua-compatiable" content="IE=edge">
  1. name属性：主要用于描述网页，与之对应属性值为content, content中的内容主要是便于搜索引擎机器人查找信息和分类信息使用的
<meta name="keywords" content="meta总结，html meta，meta属性，meta跳转">

type属性值 表现形式 对应代码

属性说明:

name：表单提交时的“键”，注意和id的区别
value：表单提交时对应项的值

type="text","password","hidden"时，为输入框的初始值
type="checkbox", "radio", "file"，为输入相关联的值
checked：radio和checkbox默认被选中的项
readonly：text和password设置只读

# 浮动的影响
浮动照成父标签塌陷(类似于将口袋里面的东西拿到了口袋的外面，原本鼓鼓的口袋，会直接塌陷)

# 补充
浮动的元素有时候也会遮挡低下的区域，如果这个区域里有文本内容，那么浏览器会遵循文本优先展示的规则将文本优先展示
  • 解决浮动造成的影响
clear: left; 规定元素左边不允许其他元素浮动
clear: right; 规定元素右边不允许其他元素浮动
推导流程：
	1. 直接写div然后写对应的长宽
	2. 写div然后添加clear属性 避免去查找长宽
	3. 万能公式(固定搭配 记住即可)
		.clearfix: after {

# 定位的状态
1. 静态定位 -- static
	所有的标签都是默认静态标签，也就是不能通过定位符操作改变位置

2. 相对定位 -- relative
	相对于原来的标签位置进行定位

3. 绝对定位 -- absolute
	相对于已经定位过得父级标签做定位

4. 固定定位 -- fixed
	相对于浏览器窗口做定位
