  • 异常处理

在写程序的过程中可能会报出很多异常，而异常报出之后程序就会停止运行，当时有部分异常是对于用户来说是非致命的，所以这里就需要对一些可能出现的异常进行预测解决方案

  • 常见的异常类型
1. SyntexError  # 语法错误
2. indexError   # 索引错误
3. nameError    # 名字错误
4. keyError     # 键错误
5. IndetationError  # 缩进错误
  • 异常的语法结构

首先来看一下异常的语法结构

#  单异常解决
try:
		待监测的代码
except '接收的异常类型' as e:
		解决异常的方案

#   多异常解决，多异常解决和单异常是一样的，只是多放了几个异常类型
try:
		待监测的代码
except '第一种异常类型' as e:
		'解决方案'
except '第二种异常类型' as e:
		'解决方案'

#   万能异常，万能异常可以用于接收任何的异常，常用的万能异常接收有Exception和BaseException，这两个可以接收所有的异常

try:
		待检测的代码
except Exception e:
		'解决方案'
		
#   try 异常还可以配合else使用，在执行中没有异常else才被执行，
try:
	待监测的代码
except '异常类型' as e:
	解决方案
else:
		print('上面全部没有异常我才会执行')
		
# finally 在使用了finally，无论上面的是否执行finally都会执行

try:
		待监测的代码
except '异常类型' as e:
		解决方案
else:
		print(解决方案)

finally:
		print('上面都执行完了，现在我再执行')

  • 生成器对象
  1. 生成器对象本质是具有__iter__和__next__方法的迭代器对象
  2. 两者的区别是，一个是内置的python解释器提供给我们而生成器是程序员自己写的， 实质的区别：
  3. 迭代器，python解释器提供 数据类型>文件对象>迭代器对象
  4. 生成器，程序员提供，代码>关键字>生成器对象
  • 生成器语法
# 生成器语法就是在写函数时添加yield关键字
def func_name()
	print('这个是人为提供的生成器')
	yield

# 这里需要注意，如果函数体中有yield关键字，我们在第一次调用的时候不会直接被调用出来，而是会生成一个生成器对象，相当于迭代器对象
# 演示代码如下
def func():
    print('这是一个生成器对象')
    yield

res = func()
res.__next__()
这是一个生成器对象
None
# 当第一次调用时，函数遇到yield就会变形为生成器对象，当第二次调用时，会调用这个生成器对象，被打印其中的内容，需要注意的是，每一次调用之后就会停留在第二个yield，并等待下一次调用
  • yield的冷门用法
# 在使用的过程中可以将yield赋予变量，并且给变量名使用send()插入数据

补充：索引取值和迭代器取值的区别

  1. 索引取值，索引取值可以反复取值，并且可以在不同的位置去，但是不支持无序类型的数据取值
  2. 迭代取值，迭代取值只能从前向后取，并且只能取一次，支持无序类型的数据取值
  • 什么是模块

模块就是具有功能的py文件，在写代码的过程中，可以在执行文件中，调用其他文件已经写好的功能，

  • 模块的分类
  1. 自定义模块，由我们自己写的py文件，我们在调用时被称为，自定义模块
  2. 内置模块，python中有很多内置切好用的模块，这些模块是官方自己开发
  3. 第三方模块，第三方模块时由第三方的开发者大佬开发提供
  • 导入模块的两种句式
  1. import …
  2. from … import
  • import和from import 的优点和缺点
# 在使用第一种句式时，需要使用import 模块的名称， 在使用者种句式时，需要说过句点符调用模块中的功能

import parameter 优点; 在调用模块时使用句点符调用模块中的变量或者函数，名称不会混乱，缺点: 在调用时每次都需要使用句点符

form modular_name import parameter 的优缺点，优点：在调用模块时
不需要使用句点符，调用方便，缺点：由于不使用句点符调用模块中的功能，可能会导致名称冲突


# 在模块中如果导入的名字太长，不太好调用，可以使用as 从新定义一个别名
  • 模块导入之循环导入问题

循环导入就类似于python垃圾回收机制中的循环引用问题，主要问题在调用模块时这个变量或者说名字还没有被调用， 如何解决这个问题，在开发过程中我们需要尽量避免循环导入的问题，在使用时，可以在导入时把导入模块的动作放在最上面执行，或者替换一下变量存储问题来避免

时间的3种格式

  1. 时间戳 ，当前距离1970年1月1号的秒数
  2. 结构化时间 计算机容易识别的时间数据
  3. 格式化时间 人类容易识别的时间数据
  4. 时间的表示 %Y 年 %m 月 %d 日，%H 小时 %M 分钟 %S 秒 [%X]=%H%M%S

print(time.strftime('%Y-%m-%d %H:%M:%S')) # 结构化时间

# 使用ranmod写随机验证码
import random
def ranmod_num(n):
    code = ''
    for i in range(n):
        random_upnum = chr(random.randint(65, 90))
        random_lownum = chr(random.randint(97, 122))
        random_int = str(random.randint(0, 9))
        temp = random.choice([random_int, random_lownum, random_upnum])
        code += temp
    return code
res = ranmod_num(11)
print(res)
