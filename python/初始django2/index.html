  • 静态文件

什么是静态文件，一般网页上使用的不经常变化的文件属于静态文件，主要针对于css文件，js文件，img文件和其他的一些框架文件等

  • 静态文件配置 django针对静态文件资源需要单独开始一个目录统一存放，一般命名为statuc目录 static目录下可以对不同的资源文件进行分类，比如 css目录 js目录 img目录 libs目录

资源访问： 在浏览器中的地址栏中之所以可以通过url后面不同的路由找到相应的资源，是因为程序员提前开了对应的资源的接口 在Django中访问静态资源需要更改默认配置

├── django666  # 项目目录
│   ├── __init__.py
│   ├── settings.py  # 配置文件
│   ├── urls.py
│   └── wsgi.py
└── manage.py

1. 如果在Django中想使用静态文件，则需要对settings.py 文件做出对应的修改

vim settings.py

# 访问静态文件资源的接口前缀(通行证)
STATIC_URL = '/static/'
# 需要添加的配置文件，名称必须大写，不然会被默认过滤掉，并且使用逗号隔开可以写多个
STATICFILES_DIRS = [
	os.path.join(BASH_DIR, 'static')  # 存储静态文件资源的目录名称
]

2. 在接口前缀正确之后会拿着好哦吗的路径依次去上述添加的列表中，从上向下寻找，找到则返回



3. 如何动态的匹配前缀（使用Django提供的语法）
{% load static %}  # 使用关键字load定义前缀为static，调用如下
<link rel="stylesheet" href="{% static 'bootstrap-3.4.1-dist/css/bootstrap.min.css'%}">
<script src="{% static 'bootstrap-3.4.1-dist/js/bootstrap.min.js' %}"></script>
  • 补充知识 form表单
form 表单参数 action 控制数据提交地址
	1. action=''	数据默认提交给当前页面所在的地址
	2. action='https.www.baidu.com'		完整地址，数据提交至填写的完整地址
	3. action='/index/'		朝当前服务端的index地址提交

method  控制数据提交的方法，默认是get
1. 默认使用get请求，get请求可以携带参数，携带数据的方式：URL?xxx=yyy&uuu=zzz
url是前面服务前访问的地址，使用?号分隔，用表单中name传参，多个以&分割

2. post 使用post请求可以向服务端提交数据，携带数据的方式为：请求体，请求体携带数据安全性较高并且没有大小限制

需要注意Django前期未配置csrf时需要先在settings中注释 MIDDLEWARE中的'django.middleware.csrf.CsrfViewMiddleware',配置
  • request对象中的一些方法

Django传参中使用的request形产就是一个对象，该对象中集成了很多方法可以使用，接下来介绍其中的一小部分

1. request.method 获取请求方式，结果为纯大写的字符串 （GET/POST）
2. request.POST  获取post请求请求体内携带的数据
3. request.POST.get()  获取列表中的最后一个数据值
4. request.POST.getlist()  获取整个列表中的数据值
5. request.GET  获取网址问号后面携带的数据
6. request.GET.get()  获取列表最后一个数据值
7. request.GET.getlist()  获取整个列表中的数据

Django 自带了sqlite3 是一个小型的数据库 功能比较少，主要用于本地测试，实际项目中一般使用其他数据库，例如这里使用MySQL数据库作为案例

1. 在Django配置文件中默认配置数据库项为 DATABASES，默认配置如下
DATABASES = {
	'default' : {
		'ENGINE': 'django.db.backends.sqlite3',
		'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
	}
}


2. 修改配置文件为MySQL链接配置文件
DATABASES = {
	'default': {
		'ENGINE': 'django.db.backends.mysql',

		'PORT': '3306',
		'USER': 'root',
		'PASSWORD': '2123',

}
